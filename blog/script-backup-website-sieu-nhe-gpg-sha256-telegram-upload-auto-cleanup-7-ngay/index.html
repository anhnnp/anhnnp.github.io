Telegram dùng để backup cá nhân: Được, nếu bạn chấp nhận rủi ro sau

Ưu điểm

  • Dung lượng miễn phí lớn (gửi bao nhiêu file cũng được, miễn <2GB hoặc <4GB nếu có Premium).
  • Upload nhanh & ổn định, ít bị chặn.
  • Tiện dụng, không cần cấu hình server.
  • Hỗ trợ API + tool telegram-upload → tự động hóa backup khá đơn giản.
  • File gửi vào Saved Messages hoặc nhóm private là private.

Nếu bạn chỉ cần:

  • Lưu snapshot nhỏ (ảnh, config, file mã hóa…),
  • Backup phụ trợ ngoài các nền tảng chính → thì Telegram hoàn toàn dùng được.

❌ Nhưng không nên dùng Telegram như cloud backup chính cho dữ liệu quan trọng

Lý do:

1. Telegram không thiết kế như một dịch vụ lưu trữ lâu dài

Telegram có thể:

  • Thay đổi chính sách bất kỳ lúc nào.
  • Hạn chế dung lượng trong tương lai (vì không phải dịch vụ lưu trữ).
  • Xóa file cũ theo cơ chế tối ưu hệ thống (không công bố).

2. Không có cam kết dữ liệu

Không có các yếu tố quan trọng của backup chuyên nghiệp:

  • Không SLA
  • Không versioning
  • Không cơ chế chống mất dữ liệu
  • Không xác minh file integrity

3. Quyền riêng tư phụ thuộc vào Telegram

Dù nhóm private, mọi thứ phụ thuộc vào:

  • Tài khoản Telegram của bạn
  • Bảo mật thiết bị
  • Chính sách Telegram

Nếu tài khoản bị hack — toàn bộ backup cũng mất.

4. Giới hạn dung lượng file

  • Free: 2GB mỗi file
  • Premium: 4GB mỗi file → Khá hạn chế với các bản backup lớn (database, VM, image…).

Cài đặt và cấu hình công cụ telegram-upload (A–Z)

telegram-upload là một công cụ dòng lệnh (CLI) cho phép bạn tải file trực tiếp lên Telegram bằng chính tài khoản cá nhân của mình. Đây là công cụ cực nhẹ, rất phù hợp để tự động hóa quá trình sao lưu (backup) website WordPress lên Telegram.

Dưới đây là hướng dẫn chi tiết từng bước để cài và cấu hình telegram-upload trên Linux server (Ubuntu/Debian), áp dụng tương tự cho các hệ thống Linux khác hoặc WSL.

1. Cài đặt Python & pip

telegram-upload chạy trên Python 3, vì vậy bạn cần đảm bảo server có đủ Python và pip:

sudo apt update
sudo apt install -y python3 python3-pip

Kiểm tra phiên bản:

python3 --version
pip3 --version

2. Cài đặt telegram-upload

Bạn có thể cài theo hai cách, nhưng cách an toàn và phổ biến nhất là cài cho user hiện tại, tránh dùng sudo:

Cài đặt cho user (recommended)

pip3 install --user -U telegram-upload

Thêm ~/.local/bin vào PATH nếu chưa có:

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

Kiểm tra công cụ

telegram-upload --help

Nếu hiện ra danh sách lệnh là bạn đã cài thành công.

Lấy api_id và api_hash từ Telegram

telegram-upload yêu cầu bạn dùng chính tài khoản Telegram, không dùng bot. Bạn cần tạo App API trên Telegram theo hướng dẫn:

  1. Truy cập: https://my.telegram.org
  2. Đăng nhập bằng số điện thoại Telegram
  3. Chọn API Development Tools
  4. Nhập tên ứng dụng tùy thích (ví dụ: BackupWP)
  5. Sau khi tạo, bạn sẽ nhận được:
    • App api_id
    • App api_hash

Hãy lưu lại hai thông tin này vì bạn sẽ nhập vào lần cấu hình đầu tiên.

Thiết lập session lần đầu

Đây là bước để Telegram cho phép server của bạn sử dụng API upload file.

Chạy lệnh:

telegram-upload --interactive

Công cụ sẽ yêu cầu:

  1. Nhập số điện thoại (định dạng quốc tế, ví dụ +84901234567)
  2. Nhập mã xác thực được gửi vào Telegram app
  3. Nhập mật khẩu 2FA (nếu tài khoản bạn dùng 2-step verification)
  4. Nhập api_id và api_hash đã lấy ở bước trước

Sau khi hoàn tất, telegram-upload sẽ tạo file cấu hình tại:

~/.config/telegram-upload.json  
~/.config/telegram-upload.session

Lưu ý bảo mật: Hai file này cho phép server của bạn đăng nhập Telegram. Hãy giới hạn quyền truy cập:

chmod 600 ~/.config/telegram-upload.*

Kiểm tra upload thử nghiệm

Tạo một file test:

echo "hello telegram" > /tmp/test.txt

Upload vào Saved Messages:

telegram-upload /tmp/test.txt

Upload vào nhóm riêng (nếu bạn có private group để chứa backup):

telegram-upload --to "https://t.me/YOUR_PRIVATE_LINK" /tmp/test.txt

Upload và ghi lại file_id (dùng cho backup script):

telegram-upload --to "https://t.me/YOUR_PRIVATE_LINK" \
                --print-file-id \
                /tmp/test.txt

Nếu file xuất hiện trong Telegram → cài đặt thành công.

Bash script backup lên Telegram

Giả sử mọi file backup đã nén + mã hóa GPG nằm trong thư mục: /path/to/backup-local với pattern kiểu:

  • wp-2025-12-03-files.tar.gz.gpg
  • wp-2025-12-03-db.sql.tar.gz.gpg

File: backup_to_telegram.sh

#!/usr/bin/env bash
set -euo pipefail

#######################
# CONFIG
#######################

# Thư mục chứa file backup .gpg
BACKUP_DIR="/path/to/backup-local"

# Log những file đã gửi lên Telegram:
# format: filename<TAB>sha256<TAB>file_id<TAB>uploaded_at
SENT_LOG="$BACKUP_DIR/sent_log.tsv"

# Log hoạt động script
LOG_FILE="$BACKUP_DIR/telegram_backup.log"

# Chat/group backup trên Telegram
# Có thể dùng private link, username, hoặc "me"
TELEGRAM_TO="https://t.me/YOUR_PRIMARY_LINK"

# Đường dẫn binary telegram-upload
TELEGRAM_BIN="$HOME/.local/bin/telegram-upload"

# Giữ backup local tối đa bao nhiêu ngày
KEEP_DAYS=7

#######################
# FUNCTIONS
#######################

log() {
  # log ra stderr + file log
  local msg="$*"
  echo "$(date '+%Y-%m-%d %H:%M:%S') [telegram-backup] $msg" | tee -a "$LOG_FILE" >&2
}

ensure_tools() {
  if ! command -v sha256sum >/dev/null 2>&1; then
    echo "Missing sha256sum. Install coreutils (Linux) trước." >&2
    exit 1
  fi

  if ! [ -x "$TELEGRAM_BIN" ]; then
    echo "telegram-upload not found at $TELEGRAM_BIN" >&2
    echo "Install: sudo pip3 install -U telegram-upload" >&2
    exit 1
  fi
}

init_files() {
  mkdir -p "$BACKUP_DIR"
  touch "$SENT_LOG" "$LOG_FILE"
}

rotate_local_backups() {
  # Xóa file .gpg local quá KEEP_DAYS ngày
  # chỉ xóa file, không đụng sent_log / log
  find "$BACKUP_DIR" -maxdepth 1 -type f -name '*.gpg' -mtime +"$KEEP_DAYS" -print -delete \
    | sed 's/^/Deleting old local backup: /' \
    | tee -a "$LOG_FILE" >&2 || true
}

already_sent() {
  local file_name="$1"
  local sha256="$2"

  # Kiểm tra xem đã có dòng filename + sha256 trong log chưa
  grep -Fq "$file_name"$'\t'"$sha256" "$SENT_LOG"
}

upload_file_to_telegram() {
  local file="$1"
  local file_name
  file_name="$(basename "$file")"

  # 1. Hash SHA256 để đảm bảo file đọc được và log lại
  log "Calculating sha256 for $file_name ..."
  local sha256
  sha256="$(sha256sum "$file" | awk '{print $1}')"

  if already_sent "$file_name" "$sha256"; then
    log "Skip (already uploaded with same hash): $file_name"
    return 0
  fi

  # Caption có tag [WP-BACKUP] + filename + hash để sau này restore dễ
  # Dùng $'' để cho phép xuống dòng \n trong Bash
  local caption
  caption=$'[WP-BACKUP] '"$file_name"$'\nsha256: '"$sha256"

  log "Uploading $file_name to Telegram chat: $TELEGRAM_TO"

  # 2. Upload + lấy file_id (dòng cuối cùng từ stdout)
  local output
  if ! output="$("$TELEGRAM_BIN" \
        --to "$TELEGRAM_TO" \
        --print-file-id \
        --caption "$caption" \
        "$file" 2>&1)"; then
    log "ERROR while uploading $file_name"
    log "telegram-upload output:"
    log "$output"
    return 1
  fi

  # Thường output chỉ là file_id, nhưng để chắc ăn ta lấy dòng cuối cùng
  local file_id
  file_id="$(printf '%s\n' "$output" | tail -n1 | tr -d '\r')"

  log "Upload OK: $file_name (file_id: $file_id)"

  # 3. Ghi log đã gửi thành công (coi như “verified” ở mức transport)
  printf '%s\t%s\t%s\t%s\n' \
    "$file_name" \
    "$sha256" \
    "$file_id" \
    "$(date --iso-8601=seconds)" >>"$SENT_LOG"
}

#######################
# MAIN
#######################

ensure_tools
init_files
rotate_local_backups

shopt -s nullglob
backup_files=("$BACKUP_DIR"/*.gpg)

if (( ${#backup_files[@]} == 0 )); then
  log "No .gpg backup files found in $BACKUP_DIR. Nothing to upload."
  exit 0
fi

for file in "${backup_files[@]}"; do
  upload_file_to_telegram "$file"
done

log "Done sending backups to Telegram."

Script này làm được gì?

  • ✅ Hash check SHA256 trước khi gửi (sha256sum)
  • ✅ Tránh upload trùng: chỉ upload nếu (filename, sha256) chưa có trong sent_log.tsv
  • ✅ Xác nhận upload thành công:
    • nếu telegram-upload lỗi → script log lỗi & exit code ≠ 0
    • nếu ok → log file_id Telegram + timestamp
  • ✅ Xoá backup local quá 7 ngày (có thể tắt nếu bạn không muốn)

Verify ở mức sâu hơn thì làm sao?

telegram-upload không có --verify, nên không có cơ chế built in tải xuống lại để so sánh hash. Cách “best practice” mình gợi ý:

  • Caption đã ghi sẵn sha256:<hash>
  • Khi bạn cần test restore định kỳ (ví dụ 1 tuần 1 lần):
    1. Download 1 file backup từ Telegram về
    2. Chạy sha256sum file đã tải
    3. So sánh với hash trong caption

Đó là cách kiểm chứng end to end (Telegram thật sự giữ đúng bits).

Nếu bạn muốn auto verify (tải xuống lại ngay sau upload để so sánh hash) thì cũng làm được nhưng rất tốn băng thông, mình không khuyến khích cho use case backup phụ.

Script tự xóa backup Telegram cũ hơn 7 ngày

Phần này telegram-upload CLI không hỗ trợ, nên phải dùng Telethon (Python) để gọi thẳng Telegram API.

Ý tưởng:

  • Tạo 1 private group (ví dụ: WP Backup)
  • Tất cả backup gửi vào group này với caption prefix: [WP-BACKUP]
  • Python script:
    • duyệt tin nhắn trong group
    • nếu tin do chính bạn gửi (msg.out == True)
    • và caption bắt đầu bằng [WP-BACKUP]
    • và msg.date < now - 7 days → delete

Cài đặt Telethon

pip install telethon

File: cleanup_telegram_backups.py

#!/usr/bin/env python3
import os
from datetime import datetime, timedelta, timezone

from telethon import TelegramClient

# --- CONFIG từ biến môi trường ---
API_ID = int(os.environ["TG_API_ID"])          # my.telegram.org
API_HASH = os.environ["TG_API_HASH"]           # my.telegram.org
SESSION = os.environ.get("TG_SESSION", "telegram-upload")
TARGET_CHAT = os.environ["TG_BACKUP_CHAT"]     # vd: "https://t.me/your_private_link" hoặc "@yourgroup"
MAX_AGE_DAYS = int(os.environ.get("TG_MAX_AGE_DAYS", "7"))
TAG_PREFIX = os.environ.get("TG_BACKUP_TAG", "[WP-BACKUP]")

async def main():
    cutoff = datetime.now(timezone.utc) - timedelta(days=MAX_AGE_DAYS)
    deleted = 0

    async with TelegramClient(SESSION, API_ID, API_HASH) as client:
        async for msg in client.iter_messages(TARGET_CHAT):
            # Bỏ qua message không có date hoặc text
            if not msg or not msg.date or not msg.message:
                continue

            # Chỉ động vào message do chính tài khoản mình gửi + có tag backup
            if not msg.out:
                continue
            if not msg.message.startswith(TAG_PREFIX):
                continue

            if msg.date < cutoff:
                await msg.delete()
                deleted += 1

    print(f"Deleted {deleted} old backup messages (>{MAX_AGE_DAYS} days).")

if __name__ == "__main__":
    import asyncio
    asyncio.run(main())

Chạy thử

export TG_API_ID=123456
export TG_API_HASH="your_api_hash_here"
export TG_BACKUP_CHAT="https://t.me/YOUR_PRIMARY_LINK"
export TG_MAX_AGE_DAYS=7
export TG_BACKUP_TAG="[WP-BACKUP]"

python3 cleanup_telegram_backups.py

Lần chạy đầu Telethon sẽ hỏi số điện thoại + code giống như khi setup telegram-upload.

Cron gợi ý

  • Gửi backup lên Telegram mỗi ngày 1 lần (ví dụ 02:10 sáng):
10 2 * * * /path/to/backup_to_telegram.sh >/dev/null 2>&1
  • Xóa backup Telegram quá 7 ngày mỗi ngày 1 lần (ví dụ 03:00 sáng):
0 3 * * * cd /path/to/script && \
  TG_API_ID=123456 TG_API_HASH="your_api_hash" \
  TG_BACKUP_CHAT="https://t.me/YOUR_PRIMARY_LINK" \
  TG_MAX_AGE_DAYS=7 TG_BACKUP_TAG="[WP-BACKUP]" \
  /usr/bin/python3 cleanup_telegram_backups.py >>cleanup_tg.log 2>&1
  • Dùng 1 cron job, chạy backup xong là dọn luôn:
# Mỗi ngày 2:10 sáng:
10 2 * * * /path/to/backup_to_telegram.sh && \
           TG_API_ID=123456 TG_API_HASH="your_api_hash" \
           TG_BACKUP_CHAT="https://t.me/YOUR_PRIMARY_LINK" \
           TG_MAX_AGE_DAYS=7 TG_BACKUP_TAG="[WP-BACKUP]" \
           /usr/bin/python3 /path/to/cleanup_telegram_backups.py \
           >>/path/to/cleanup_tg.log 2>&1

Tóm lại

Với combo:

  • backup_to_telegram.sh
  • cleanup_telegram_backups.py

bạn đang có:

  • File backup đã mã hóa + sha256 log riêng
  • Caption trên Telegram có tag + hash → dễ kiểm tra / restore
  • Ghi log file_id để truy dấu
  • Dọn Telegram chat, luôn chỉ giữ 7 ngày backup gần nhất
  • Có thể cắm thêm rclone để sync Google Drive trong cùng workflow.
